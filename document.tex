\section{Tutorial 1: Primeiros Passos com UTM e Raspberry Pi OS}
\label{sec:Etapa 1: Primeiros Passos com UTM e Raspberry Pi OS}
    Nesta primeira etapa, o objetivo é proporcionar uma forma visual e simples de interagir com um ambiente de Raspberry Pi. O uso do \textbf{UTM} é recomendado, pois permite executar o Raspberry Pi OS dentro de uma máquina virtual através de uma interface gráfica, reduzindo a complexidade inicial. De acordo com a Figura \ref{fig:tabela da luisa}.


    \begin{figure}[h]
        \centering
        \includegraphics[scale=0.20]{experiment-01-general-architecture.pdf}
        \caption{Figura teste da Luisa}
        \label{fig:tabela da luisa}
    \end{figure}

    
    \subsubsection*{Objetivos}
    \begin{itemize}
        \item Familiarizar-se com a interface gráfica do Raspberry Pi OS.
        \item Compreender o conceito de máquinas virtuais e emulação.
        \item Aprender a iniciar, desligar e navegar no ambiente simulado do Raspberry Pi.
    \end{itemize}
    
    \subsubsection*{Requisitos}
    \begin{itemize}
        \item Computador com macOS, Windows ARM ou Linux.
        \item Conexão com a Internet para download do UTM e da imagem do Raspberry Pi OS.
    \end{itemize}
    
    \subsubsection*{Guia de Instalação}
    \begin{enumerate}
        \item Baixar e instalar o UTM:
        \begin{itemize}
            \item macOS: \url{https://mac.getutm.app}
            \item Windows/Linux: \url{https://docs.getutm.app}
        \end{itemize}
        \item Baixar uma imagem pré-compilada do Raspberry Pi OS compatível com QEMU/UTM no site oficial:
        \begin{itemize}
            \item \url{https://www.raspberrypi.com/software/operating-systems/}
        \end{itemize}
        \item Abrir o UTM e criar uma nova Máquina Virtual:
        \begin{itemize}
            \item Selecionar ``Create New Virtual Machine''.
            \item Escolher a imagem do Raspberry Pi OS baixada.
            \item Configurar a memória (mínimo recomendado: 2 GB).
            \item Configurar o tamanho do disco virtual (mínimo recomendado: 8 GB).
        \end{itemize}
        \item Iniciar a Máquina Virtual e aguardar a inicialização do Raspberry Pi OS.
    \end{enumerate}
    
    \subsubsection*{Primeiras Interações}
    \begin{itemize}
        \item Fazer login com as credenciais padrão (normalmente \texttt{pi} / \texttt{raspberry}).
        \item Explorar o ambiente de desktop (gerenciador de arquivos, navegador, terminal).
        \item Praticar as operações básicas de reinício e desligamento da VM.
    \end{itemize}
    
    \subsubsection*{Resultados de Aprendizagem Esperados}
    \begin{itemize}
        \item Entender o que é um ambiente virtualizado de Raspberry Pi.
        \item Ganhar confiança na navegação pelo sistema Raspberry Pi OS via interface gráfica.
        \item Estar preparada para a próxima etapa, que introduzirá o uso do terminal e a instalação de pacotes.
    \end{itemize}

\section{Tutorial 2: Uso do Terminal no Raspberry Pi OS via UTM}
    \label{sec: Etapa 2: Uso do Terminal no Raspberry Pi OS via UTM}
        Após a familiarização inicial com a interface gráfica (Etapa 1), esta segunda etapa tem como objetivo introduzir o uso do terminal Linux no ambiente do Raspberry Pi OS. A ideia é que a aluna aprenda a executar comandos básicos, instalar pacotes e rodar scripts em Python diretamente pela linha de comando, ganhando autonomia no controle do sistema.
        
        \subsubsection*{Objetivos}
        \begin{itemize}
            \item Aprender a abrir e utilizar o terminal do Raspberry Pi OS.
            \item Executar comandos básicos de navegação e gerenciamento de arquivos.
            \item Instalar pacotes utilizando o gerenciador \texttt{apt}.
            \item Criar e executar scripts simples em Python.
        \end{itemize}
        
        \subsubsection*{Requisitos}
        \begin{itemize}
            \item Ter concluído a Etapa 1 (UTM configurado com Raspberry Pi OS em funcionamento).
            \item Conexão com a Internet dentro do Raspberry Pi OS (para instalar pacotes).
        \end{itemize}
        
        \subsubsection*{Guia de Aprendizagem}
        \begin{enumerate}
            \item Abrir o terminal no Raspberry Pi OS (ícone no menu ou atalho de teclado).
            \item Praticar comandos básicos:
            \begin{itemize}
                \item \texttt{ls} – listar arquivos e diretórios.
                \item \texttt{cd <diretório>} – navegar entre pastas.
                \item \texttt{mkdir <nome>} – criar uma nova pasta.
                \item \texttt{rm <arquivo>} – remover arquivos.
            \end{itemize}
            \item Atualizar os pacotes do sistema:
            \begin{verbatim}
            sudo apt-get update
            sudo apt-get upgrade
            \end{verbatim}
            \item Instalar o Python e pacotes auxiliares (se não estiverem presentes):
            \begin{verbatim}
            sudo apt-get install python3 python3-pip -y
            \end{verbatim}
            \item Criar um arquivo Python simples:
            \begin{verbatim}
            nano ola.py
            \end{verbatim}
            Inserir o código:
            \begin{verbatim}
            print("Olá, mundo do Raspberry Pi!")
            \end{verbatim}
            Salvar (\texttt{CTRL+O}) e sair (\texttt{CTRL+X}).
            \item Executar o arquivo criado:
            \begin{verbatim}
            python3 ola.py
            \end{verbatim}
        \end{enumerate}
        
        \subsubsection*{Primeiros Exercícios Práticos}
        \begin{itemize}
            \item Criar scripts em Python que exibam mensagens personalizadas.
            \item Instalar a biblioteca \texttt{pygame} e testar sons básicos:
            \begin{verbatim}
            pip3 install pygame
            \end{verbatim}
            \item Criar um script que reproduza um som curto usando \texttt{pygame}.
        \end{itemize}
        
        \subsubsection*{Resultados de Aprendizagem Esperados}
        \begin{itemize}
            \item Entender o funcionamento do terminal Linux dentro do Raspberry Pi OS.
            \item Adquirir noções básicas de gerenciamento de arquivos e pacotes.
            \item Rodar scripts em Python para experimentar funções relacionadas a som e interação.
            \item Preparar-se para a Etapa 3, onde será introduzido o uso de Docker e ambientes de desenvolvimento mais avançados.
        \end{itemize}

\section{Tutorial 3: Design do Jogo e Feedback Musical}
\label{sec: Etapa 3: Design do Jogo e Feedback Musical}

    Esta etapa tem como objetivo planejar a lógica principal do jogo, definir os elementos de feedback musical que serão utilizados e iniciar a prototipação visual da interface. O foco aqui é unir elementos de usabilidade com estímulos auditivos que favoreçam a atenção da criança durante a tarefa. 

    \textbf{{Texto de exemplo para ilustrar referências:}} seguindo os conceitos de \cite{franco2014vantagens} e de \cite{roque2017inserccao} os passos para criar um jogo são blabablabla.... 
    
    \subsubsection*{Objetivos}
    \begin{itemize}
        \item Planejar o fluxo básico do jogo com estímulos musicais.
        \item Selecionar sons curtos e lúdicos que possam atuar como feedback positivo ou negativo.
        \item Criar um esboço visual da interface do jogo, mesmo que em papel ou ferramenta online.
    \end{itemize}
    
    \subsubsection*{Requisitos}
    \begin{itemize}
        \item Ter definido o público-alvo e o tipo de tarefa (ex: identificação de sons por timbre, altura, etc.).
        \item Conhecimentos básicos sobre lógica de jogos e elementos visuais.
    \end{itemize}
    
    \subsubsection*{Guia de Aprendizagem}
    \begin{enumerate}
        \item Esboçar a lógica central do jogo:
        \begin{itemize}
            \item Como a criança interage com o sistema (clique? teclado? resposta por tempo?).
            \item O que acontece ao acertar ou errar.
            \item Como será a progressão da dificuldade.
        \end{itemize}
        \item Escolher os tipos de som:
        \begin{itemize}
            \item Som de acerto, erro, início de rodada, fim de jogo.
            \item Usar sites como \texttt{freesound.org} ou gerar sons simples em Python.
        \end{itemize}
        \item Criar um protótipo visual simples:
        \begin{itemize}
            \item Pode ser feito em papel, PowerPoint, Figma, Canva ou outros.
            \item Marcar claramente os botões, áreas de estímulo, e o retorno sonoro.
        \end{itemize}
    \end{enumerate}
    
    \subsubsection*{Primeiros Exercícios Práticos}
    \begin{itemize}
        \item Criar um fluxograma da lógica de interação.
        \item Baixar pelo menos três sons para testar.
        \item Fazer uma versão "printável" ou digital do layout da interface.
    \end{itemize}
    
    \subsubsection*{Resultados de Aprendizagem Esperados}
    \begin{itemize}
        \item Ter clareza sobre como o jogo vai funcionar do ponto de vista da criança.
        \item Ter um conjunto de sons prontos para uso e testados previamente.
        \item Ter um protótipo que possa guiar a implementação na Etapa 3.
    \end{itemize}

\section{Tutorial 5: Estrutura Básica do Jogo (Pygame)}
\label{sec: Etapa 5: Estrutura Básica do Jogo (Pygame)}

    Nesta etapa inicia-se a implementação do jogo \textit{Musicalizando no Céu} em Python, utilizando a biblioteca \texttt{pygame}. O objetivo é criar a janela, o laço principal de execução, o fundo (céu) e um primeiro estímulo visual (estrela piscando).
    
    \subsubsection*{Objetivos}
    \begin{itemize}
        \item Configurar um ambiente de projeto em Python (preferencialmente com \textit{virtualenv}).
        \item Instalar e validar o \texttt{pygame}.
        \item Renderizar uma janela com fundo e uma estrela piscando.
    \end{itemize}
    
    \subsubsection*{Requisitos}
    \begin{itemize}
        \item Raspberry Pi OS funcional (via VirtualBox).
        \item Python 3 instalado.
        \item Acesso à internet ou mídia local com os pacotes.
    \end{itemize}
    
    \subsubsection*{Guia de Aprendizagem}
    \begin{enumerate}
        \item Abrir o terminal no Raspberry Pi OS e criar a pasta do projeto:
    \begin{verbatim}
    mkdir -p ~/projetos/musicalizando-no-ceu && cd ~/projetos/musicalizando-no-ceu
    \end{verbatim}
    
        \item (Opcional, recomendado) Criar e ativar um ambiente virtual:
    \begin{verbatim}
    python3 -m venv .venv
    source .venv/bin/activate
    \end{verbatim}
    
        \item Instalar o \texttt{pygame}:
    \begin{verbatim}
    pip3 install pygame
    \end{verbatim}
    
        \item Criar a estrutura inicial de pastas e um arquivo principal:
    \begin{verbatim}
    mkdir assets assets/sounds assets/images
    touch main.py
    \end{verbatim}
    
        \item (Opcional) Adicionar uma imagem simples de estrela em \texttt{assets/images/estrela.png}.
              Caso não tenha uma imagem, o código desenhará a estrela como forma geométrica.
    
        \item Abrir o arquivo \texttt{main.py} no editor e inserir o código base:
    \begin{verbatim}
    import pygame
    import sys
    
    WIDTH, HEIGHT = 800, 480
    FPS = 60
    
    def desenhar_estrela(surface, x, y, size, color):
        # Desenha uma estrela simples via polygon (fallback sem imagem)
        pts = [(x, y - size), (x + size*0.3, y - size*0.3), (x + size, y - size*0.3),
               (x + size*0.45, y + size*0.1), (x + size*0.6, y + size),
               (x, y + size*0.45), (x - size*0.6, y + size),
               (x - size*0.45, y + size*0.1), (x - size, y - size*0.3),
               (x - size*0.3, y - size*0.3)]
        pygame.draw.polygon(surface, color, pts)
    
    def main():
        pygame.init()
        screen = pygame.display.set_mode((WIDTH, HEIGHT))
        pygame.display.set_caption("Musicalizando no Céu - Protótipo")
        clock = pygame.time.Clock()
    
        # Fundo (céu)
        SKY = (20, 30, 80)
    
        # Controle de piscar
        blink_timer = 0.0
        blink_period = 0.8  # segundos
        star_visible = True
    
        # Se houver imagem:
        star_img = None
        try:
            star_img = pygame.image.load("assets/images/estrela.png").convert_alpha()
        except Exception:
            star_img = None
    
        while True:
            dt = clock.tick(FPS) / 1000.0  # delta time em segundos
    
            for event in pygame.event.get():
                if event.type == pygame.QUIT:
                    pygame.quit()
                    sys.exit()
    
            blink_timer += dt
            if blink_timer >= blink_period:
                star_visible = not star_visible
                blink_timer = 0.0
    
            screen.fill(SKY)
    
            # Desenha estrela
            if star_visible:
                if star_img:
                    rect = star_img.get_rect(center=(WIDTH//2, HEIGHT//2))
                    screen.blit(star_img, rect)
                else:
                    desenhar_estrela(screen, WIDTH//2, HEIGHT//2, 30, (255, 215, 0))
    
            pygame.display.flip()
    
    if __name__ == "__main__":
        main()
    \end{verbatim}
    
        \item Executar o protótipo:
    \begin{verbatim}
    python3 main.py
    \end{verbatim}
    \end{enumerate}
    
    \subsubsection*{Primeiros Exercícios Práticos}
    \begin{itemize}
        \item Alterar o tamanho e a velocidade do piscar da estrela.
        \item Trocar o fundo por um gradiente de céu ou imagem.

    \end{itemize}
    
    \subsubsection*{Resultados de Aprendizagem Esperados}

    \end{itemize}

    WHITE = (245, 245, 245)
    GOLD = (255, 215, 0)
    RED  = (200, 60, 60)
    GREEN= (60, 180, 75)
    
    def desenhar_estrela(surface, x, y, size, color):
        pts = [(x, y - size), (x + size*0.3, y - size*0.3), (x + size, y - size*0.3),

        pygame.draw.polygon(surface, color, pts)
    
    def main():

        pygame.mixer.init()
        screen = pygame.display.set_mode((WIDTH, HEIGHT))
        pygame.display.set_caption("Musicalizando no Céu - Interação")

\section{Tutorial 7: Fases, Sequências e Progressão}
\label{sec: Etapa 7: Fases, Sequências e Progressão}

    Nesta etapa organizamos a lógica de fases, adicionamos sequências (listas de estímulos) e tempo-limite de resposta. Ao completar a sequência, a fase é concluída e pode-se avançar.
    
    \subsubsection*{Objetivos}
    \begin{itemize}
        \item Definir fases com parâmetros (nº de estímulos, tempo de resposta, velocidade).
        \item Implementar sequência de alvos e avaliação de acerto dentro do tempo.
        \item Exibir barra de progresso ou contador de acertos da fase.
    \end{itemize}
    
    \subsubsection*{Requisitos}
    \begin{itemize}
        \item Concluir as Etapas 5 e 6.
    \end{itemize}
    
    \subsubsection*{Guia de Aprendizagem}
    \begin{enumerate}
        \item Criar um arquivo de configuração simples (\texttt{levels.py}) com as fases:
    \begin{verbatim}
    # levels.py
    LEVELS = [
        {"name": "Fase 1", "seq_len": 4, "blink_period": 0.9, "timeout": 2.8},
        {"name": "Fase 2", "seq_len": 6, "blink_period": 0.7, "timeout": 2.0},
        {"name": "Fase 3", "seq_len": 8, "blink_period": 0.55, "timeout": 1.5},
    ]
    \end{verbatim}
    
        \item Atualizar \texttt{main.py} para controlar sequência, tempo e progresso:
    \begin{verbatim}
    import pygame, sys, random, time
    from levels import LEVELS
    
    WIDTH, HEIGHT = 800, 480
    FPS = 60
    SKY = (20, 30, 80)
    WHITE = (245, 245, 245)
    GOLD = (255, 215, 0)
    GREEN= (60, 180, 75)
    RED  = (200, 60, 60)
    
    def desenhar_estrela(surface, x, y, size, color):
        pts = [(x, y - size), (x + size*0.3, y - size*0.3), (x + size, y - size*0.3),
               (x + size*0.45, y + size*0.1), (x + size*0.6, y + size),
               (x, y + size*0.45), (x - size*0.6, y + size),
               (x - size*0.45, y + size*0.1), (x - size, y - size*0.3),
               (x - size*0.3, y - size*0.3)]
        pygame.draw.polygon(surface, color, pts)
    
    def main():
        pygame.init(); pygame.mixer.init()
        screen = pygame.display.set_mode((WIDTH, HEIGHT))
        pygame.display.setcaption = pygame.display.set_caption
        pygame.display.setcaption("Musicalizando no Céu - Fases")
        clock = pygame.time.Clock()
        font = pygame.font.SysFont(None, 26)
    
        acerto_snd = pygame.mixer.Sound("assets/sounds/acerto.wav")
        erro_snd   = pygame.mixer.Sound("assets/sounds/erro.wav")
    
        level_idx = 0
        pontos = 0
    
        while level_idx < len(LEVELS):
            level = LEVELS[level_idx]
            seq_len    = level["seq_len"]
            blink_per  = level["blink_period"]
            timeout_s  = level["timeout"]
    
            # Gera sequência (ex.: 1 = estrela acesa, 0 = apagada).
            # A regra: responder (tecla 'A' ou clique) SOMENTE quando estiver acesa.
            sequence = [random.choice([0,1]) for _ in range(seq_len)]
    
            seq_pos = 0
            star_visible = False
            blink_timer = 0.0
            feedback_color = None; feedback_timer = 0.0
    
            # Timer p/ resposta do estímulo atual
            stim_timer = 0.0
            responded  = False
            correct    = False
    
            running_phase = True
            while running_phase:
                dt = clock.tick(FPS)/1000.0
                for event in pygame.event.get():
                    if event.type == pygame.QUIT:
                        pygame.quit(); sys.exit()
                    if event.type == pygame.KEYDOWN and event.key == pygame.K_a:
                        if not responded:
                            responded = True
                            if star_visible:
                                acerto_snd.play(); pontos += 1
                                feedback_color = GREEN
                            else:
                                erro_snd.play(); pontos = max(0, pontos-1)
                                feedback_color = RED
                            feedback_timer = 0.3
                    if event.type == pygame.MOUSEBUTTONDOWN and event.button == 1:
                        if not responded:
                            responded = True
                            if star_visible:
                                acerto_snd.play(); pontos += 1
                                feedback_color = GREEN
                            else:
                                erro_snd.play(); pontos = max(0, pontos-1)
                                feedback_color = RED
                            feedback_timer = 0.3
    
                # Atualiza piscar e estímulo baseado na sequência
                blink_timer += dt
                if blink_timer >= blink_per:
                    blink_timer = 0.0
                    # novo estado visual conforme a sequência
                    if seq_pos < seq_len:
                        star_visible = bool(sequence[seq_pos])
                        seq_pos += 1
                        responded = False
                        stim_timer = 0.0
    
                # Timeout por estímulo
                if not responded and seq_pos>0 and seq_pos<=seq_len:
                    stim_timer += dt
                    if stim_timer >= timeout_s:
                        # Não respondeu a tempo: conta como erro se estrela estava acesa
                        if star_visible:
                            erro_snd.play(); pontos = max(0, pontos-1)
                            feedback_color = RED; feedback_timer = 0.3
                        responded = True  # trava até o próximo estímulo
    
                # Fim da fase quando a sequência toda foi mostrada e último estímulo expirou
                if seq_pos >= seq_len and responded:
                    running_phase = False
    
                # Desenho
                screen.fill(SKY)
                if star_visible:
                    desenhar_estrela(screen, WIDTH//2, HEIGHT//2, 30, GOLD)
                else:
                    desenhar_estrela(screen, WIDTH//2, HEIGHT//2, 30, WHITE)
    
                # Barra de progresso
                prog_w = int((seq_pos/seq_len) * (WIDTH-40))
                pygame.draw.rect(screen, WHITE, (20, 20, WIDTH-40, 10), 1)
                pygame.draw.rect(screen, GREEN, (20, 20, prog_w, 10))
    
                # Cabeçalho
                header = font.render(
                    f"{level['name']}  |  Estímulo {min(seq_pos,seq_len)}/{seq_len}  |  Pontos: {pontos}",
                    True, WHITE)
                screen.blit(header, (20, 40))
    
                # Feedback de borda
                if feedback_color:
                    pygame.draw.rect(screen, feedback_color, (0, 0, WIDTH, 5))
                    feedback_timer -= dt
                    if feedback_timer <= 0: feedback_color = None
    
                pygame.display.flip()
    
            # Animação breve de fim de fase
            t0 = time.time()
            while time.time() - t0 < 0.8:
                dt = clock.tick(FPS)/1000.0
                screen.fill(SKY)

\section{Tutorial 10: Interação com Botões — GPIO físico e simulação}
\label{sec:Tutorial10_Botoes}

Este tutorial mostra como integrar botões físicos do Raspberry Pi (GPIO) ao jogo em Python e também como oferecer alternativas de simulação (botão na tela ou mock GPIO) para desenvolvimento em PC/VirtualBox. O objetivo é permitir que a aluna controle o fluxo do jogo (ex.: iniciar fase, responder estímulo) usando: (a) botões físicos no Raspberry Pi; (b) botões na tela (Pygame); (c) um mock de GPIO para testes locais.

\subsubsection*{Objetivos}
\begin{itemize}
    \item Conectar um botão físico ao Raspberry Pi e ler seu estado a partir de Python (\texttt{gpiozero}).
    \item Implementar um botão na tela (Pygame) que simule o botão físico, útil para testes sem hardware.
    \item Demonstrar como usar \texttt{gpiozero}'s \texttt{MockFactory} para testes automatizados em PC/VirtualBox.
    \item Integrar a leitura do botão ao fluxo do jogo (ex.: mapear botão a \texttt{Começar} ou a responder estímulos na Fase 1).
\end{itemize}

\subsubsection*{Requisitos}
\begin{itemize}
    \item Raspberry Pi com GPIO acessível (se for usar hardware). 
    \item Python 3 instalado.
    \item Biblioteca \texttt{gpiozero} e \texttt{pygame} instaladas:
    \begin{verbatim}
    pip3 install gpiozero pygame
    \end{verbatim}
    \item Caso esteja no VirtualBox/PC: apenas o botão na tela ou o Mock GPIO serão usados.
\end{itemize}

\subsubsection*{Notas de segurança e permissões}
\begin{itemize}
    \item Ao usar GPIO no Raspberry Pi, execute o script com permissões adequadas (\texttt{sudo} ou grupo \texttt{gpio}).
    \item Sempre use apenas 3.3V nos GPIOs (não use 5V).
    \item Botões simples devem ser conectados entre o pino GPIO e GND, com pull-up interno habilitado.
\end{itemize}

\subsubsection*{Guia de Aprendizagem}
\begin{enumerate}
    \item Exemplo de botão físico com \texttt{gpiozero}:
    \begin{verbatim}
    # gpio_button_example.py
    from gpiozero import Button

    def on_hw_button():
        print("Botão físico pressionado! -> disparar ação do jogo")

    btn = Button(17, pull_up=True, bounce_time=0.05)  # GPIO17
    btn.when_pressed = on_hw_button

    import signal
    signal.pause()
    \end{verbatim}

    \item Exemplo de botão na tela (Pygame):
    \begin{verbatim}
    # pygame_button_example.py
    button_rect = pygame.Rect(20, 20, 140, 48)

    def on_screen_button():
        print("Botão de tela pressionado! -> disparar ação do jogo")

    for event in pygame.event.get():
        if event.type == pygame.MOUSEBUTTONDOWN and button_rect.collidepoint(event.pos):
            on_screen_button()

    pygame.draw.rect(screen, (100,180,255), button_rect, border_radius=8)
    lbl = FONT.render("BOTÃO", True, (10,10,20))
    screen.blit(lbl, (button_rect.x+10, button_rect.y+10))
    \end{verbatim}

    \item Exemplo de Mock GPIO (para testes locais):
    \begin{verbatim}
    from gpiozero import Device, Button
    from gpiozero.pins.mock import MockFactory
    import time

    Device.pin_factory = MockFactory()
    btn = Button(17, pull_up=True)

    def on_mock():
        print("Mock: botão acionado")

    btn.when_pressed = on_mock

    pin = Device.pin_factory.pin(17)
    pin.drive_low(); time.sleep(0.2); pin.drive_high()
    \end{verbatim}
\end{enumerate}

\subsubsection*{Checklist de Validação}
\begin{itemize}
    \item [ ] Botão físico (GPIO17) dispara a função esperada quando pressionado.
    \item [ ] Botão na tela é clicável e dispara a mesma ação.
    \item [ ] MockFactory permite simulação de pressões em PC/VirtualBox.
\end{itemize}

\subsubsection*{Exercícios Práticos}
\begin{itemize}
    \item Adicionar um segundo botão para ``Pausar'' o jogo.
    \item Registrar timestamps de pressões em CSV (tempo de reação).
    \item Testar MockFactory para simular diferentes padrões de resposta.
\end{itemize}

\subsubsection*{Resultados de Aprendizagem Esperados}
\begin{itemize}
    \item Integrar hardware simples (botões) com software em Python.
    \item Criar alternativa de simulação para desenvolvimento sem hardware.
    \item Garantir práticas seguras ao mexer com GPIO (uso de pull-up e tensão correta).

        def pos_rel(x_frac, y_frac, width, height):

        \end{verbatim}

        \begin{verbatim}
        positions = {
            pygame.K_LEFT:  pos_rel(0.25, 0.5, WIDTH, HEIGHT),
            pygame.K_UP:    pos_rel(0.5, 0.3, WIDTH, HEIGHT),
            pygame.K_RIGHT: pos_rel(0.75, 0.5, WIDTH, HEIGHT),
            pygame.K_DOWN:  pos_rel(0.5, 0.7, WIDTH, HEIGHT),
        }
        \end{verbatim}

\section{Tutorial 13: Introdução ao BeepBox e Recriação do Tema do Super Mario}
\label{sec:tutorial13}

Este tutorial marca o início da etapa musical do projeto, apresentando a ferramenta \textbf{BeepBox}, uma plataforma online gratuita para composição de trilhas sonoras em estilo 8-bit (chiptune). O objetivo é compreender os elementos básicos de criação musical digital e reproduzir uma versão simplificada do tema clássico do \textit{Super Mario Bros.}, aprendendo conceitos fundamentais de melodia, ritmo e instrumentação digital.

\subsubsection*{Objetivos}
\begin{itemize}
    \item Explorar a interface e os recursos básicos do BeepBox.
    \item Entender a estrutura de uma trilha sonora (melodia, acompanhamento, ritmo).
    \item Recriar o tema do \textit{Super Mario Bros.} em versão simplificada.
    \item Exportar o resultado como arquivo de áudio para uso posterior no jogo.
\end{itemize}

\subsubsection*{Materiais Necessários}
\begin{itemize}
    \item Acesso à internet e navegador web (https://www.beepbox.co).
    \item Fones de ouvido ou caixas de som.
    \item Arquivo de referência do tema do Super Mario (opcional, disponível no YouTube ou Spotify).
\end{itemize}

\subsubsection*{Passo a Passo}

\paragraph{1. Acessando o BeepBox}
Abra o site \href{https://www.beepbox.co}{https://www.beepbox.co}.  
A interface principal mostra uma linha do tempo (horizontal) e trilhas verticais correspondentes a diferentes canais de som (melodia, harmonia, baixo, percussão).

\paragraph{2. Entendendo os canais}
Cada linha colorida representa um canal:
\begin{itemize}
    \item Azul – Melodia principal (instrumento 1)
    \item Verde – Acompanhamento (instrumento 2)
    \item Amarelo – Baixo
    \item Vermelho – Percussão
\end{itemize}

\paragraph{3. Criando a melodia}
Clique sobre os quadradinhos azuis na linha do tempo para criar notas.  
Para reproduzir o tema inicial do \textit{Super Mario Bros.}, use a sequência (em dó maior, simplificada):

\begin{verbatim}
E E - E - C E - G - - - G
C - G - E - A B A A - -
\end{verbatim}

Dica: pressione a tecla \textbf{Q} para tocar a música.

\paragraph{4. Ajustando tempo e instrumento}
Na barra superior:
\begin{itemize}
    \item Ajuste o \textbf{tempo (BPM)} para 120.
    \item Selecione o instrumento “square” ou “sine” para som 8-bit autêntico.
\end{itemize}

\paragraph{5. Adicionando o ritmo (percussão)}
Ative o canal vermelho e adicione batidas regulares em 1, 3, 5, 7, criando uma base rítmica simples.

\paragraph{6. Exportando a trilha}
Clique em \textbf{File → Export → WAV} para salvar o áudio.  
Este arquivo poderá ser reproduzido dentro do jogo Python via \texttt{pygame.mixer}.

\subsubsection*{Desafio}
Tente adicionar uma segunda parte à música, repetindo a melodia em uma oitava acima e variando o ritmo de acompanhamento. Observe como pequenas mudanças alteram a emoção transmitida pela música.

\subsubsection*{Observações}
\begin{itemize}
    \item O BeepBox salva automaticamente o link com as notas — copie-o para reutilizar depois.
    \item Use fones para perceber melhor os detalhes da melodia e da percussão.
    \item Essa atividade pode ser feita em grupo: um aluno cria a melodia, outro o ritmo, e outro o acompanhamento.
\end{itemize}

\section{Tutorial: Fluxo de Trabalho com Git e GitHub}
\label{sec:fluxo-trabalho-git}

Este tutorial descreve o fluxo de trabalho padrão adotado para o desenvolvimento colaborativo do jogo musical no projeto \textit{COTB 2026}.
O objetivo é orientar o processo de edição, versionamento e atualização do código-fonte utilizando Git e GitHub.

\subsection{Visão Geral}

O controle de versão é essencial para garantir que as modificações realizadas no código do jogo sejam registradas de forma organizada e segura.
O repositório Git permite:
\begin{itemize}
    \item Manter o histórico de todas as alterações realizadas.
    \item Trabalhar em equipe sem sobrescrever o código do outro.
    \item Restaurar versões anteriores em caso de erro.
    \item Sincronizar o projeto entre diferentes computadores.
\end{itemize}

\subsection{Fluxo de Trabalho Padrão}

O processo de trabalho segue as etapas descritas abaixo:

\begin{enumerate}
    \item \textbf{Editar o código localmente:}
    \begin{itemize}
        \item Abrir o projeto no diretório local (\texttt{game-luisa/}).
        \item Fazer as modificações desejadas (ex.: adicionar novas fases, imagens, sons ou ajustes de lógica).
    \end{itemize}

    \item \textbf{Verificar alterações:}
    \begin{minted}[fontsize=\small, bgcolor=lightgray!10]{bash}
git status
    \end{minted}
    Esse comando lista os arquivos modificados, adicionados ou removidos desde o último commit.

    \item \textbf{Adicionar arquivos ao controle de versão:}
    \begin{minted}[fontsize=\small, bgcolor=lightgray!10]{bash}
git add .
    \end{minted}
    O ponto (\texttt{.}) adiciona todas as mudanças detectadas.  
    Para adicionar um arquivo específico, use:
    \begin{minted}[fontsize=\small, bgcolor=lightgray!10]{bash}
git add fase1.py
    \end{minted}

    \item \textbf{Criar um commit (salvar a versão local):}
    \begin{minted}[fontsize=\small, bgcolor=lightgray!10]{bash}
git commit -m "Mensagem descritiva sobre a mudança"
    \end{minted}
    Exemplo:
    \begin{minted}[fontsize=\small, bgcolor=lightgray!10]{bash}
git commit -m "Adiciona fade sonoro e transição visual nas fases"
    \end{minted}

    \item \textbf{Enviar as alterações para o GitHub:}
    \begin{minted}[fontsize=\small, bgcolor=lightgray!10]{bash}
git push origin main
    \end{minted}
    Esse comando envia o commit local para o repositório remoto hospedado no GitHub.

    \item \textbf{Atualizar o código local (caso outro colaborador tenha feito mudanças):}
    \begin{minted}[fontsize=\small, bgcolor=lightgray!10]{bash}
git pull origin main
    \end{minted}
    Esse comando sincroniza o repositório local com a versão mais recente publicada no GitHub.
\end{enumerate}

\subsection{Resumo do Fluxo de Comandos}

\begin{center}
\begin{tabular}{|l|l|}

\textbf{Etapa} & \textbf{Comando} \\
\hline

Adicionar arquivos & \texttt{git add .} \\
Criar commit & \texttt{git commit -m "mensagem"} \\
Enviar para o GitHub & \texttt{git push origin main} \\
Atualizar a partir do remoto & \texttt{git pull origin main} \\

\subsection{Boas Práticas}

\begin{itemize}
    \item Sempre realizar um \texttt{git pull origin main} antes de iniciar novas modificações.

\section{Tutorial 14: Acesso Remoto ao Raspberry Pi via Cabo RJ45 e SSH}
\label{sec: Tutorial 14: Acesso Remoto ao Raspberry Pi via Cabo RJ45 e SSH}
    
    Este tutorial tem como objetivo permitir o acesso remoto ao Raspberry Pi 4 utilizando um cabo de rede (RJ45) e o protocolo SSH.  
    O procedimento possibilita estabelecer comunicação direta entre o Raspberry Pi e o computador, mesmo sem roteador ou conexão Wi-Fi, permitindo diagnosticar e configurar o dispositivo em rede local.
    
    \subsection*{Objetivos}
    \begin{itemize}
        \item Testar a comunicação via cabo RJ45 com o Raspberry Pi.
        \item Estabelecer acesso remoto via SSH em diferentes cenários (DHCP ou conexão direta).
        \item Ativar o serviço SSH em sistemas já instalados, sem necessidade de reinstalação.
    \end{itemize}
    
    \subsection*{Materiais Necessários}
        \begin{itemize}
            \item Raspberry Pi 4 (com cartão microSD já configurado com o sistema operacional).
            \item Cabo RJ45 (Ethernet).
            \item Fonte USB-C de 5 V / 3 A (pode ser usada a do Samsung S24+).
            \item Computador com Linux ou macOS (para os comandos de teste).
        \end{itemize}
        
    \subsection{Opção A – Conexão via Roteador (DHCP Automático)}
        Esta é a forma mais simples de detectar o Raspberry Pi na rede.
        
        \begin{enumerate}
            \item Conecte o Raspberry Pi e o computador ao mesmo roteador (via cabo ou Wi-Fi).
            \item O roteador atribuirá um endereço IP automaticamente (DHCP).
            \item Descubra o IP do Raspberry Pi executando:
            \begin{minted}{bash}
            arp -a
            # ou
            sudo nmap -sn 192.168.0.0/24
            \end{minted}
            \item Procure por um nome como \texttt{raspberrypi.lan}.
            \item Acesse via SSH:
            \begin{minted}{bash}
            ssh pi@raspberrypi.local
            # ou usando o IP encontrado
            ssh pi@192.168.0.xxx
            \end{minted}
        \end{enumerate}
    
    \subsection{Opção B – Conexão Direta entre Raspberry Pi e Computador}
    
        Quando não há roteador disponível, o Raspberry Pi pode se comunicar por \textbf{endereçamento automático (Zeroconf)}.
        
        \begin{enumerate}
            \item Conecte o cabo RJ45 diretamente entre o Raspberry Pi e o computador.
            \item Aguarde cerca de 1 minuto para que ambos obtenham endereços do tipo \texttt{169.254.x.x}.
            \item No computador, teste:
            \begin{minted}{bash}
            ping raspberrypi.local
            ssh pi@raspberrypi.local
            \end{minted}
            \item Caso o SSH responda, o acesso está ativo.
        \end{enumerate}
        
        Se o SSH não estiver habilitado, siga a próxima subseção.


    
    \subsection{Ativando o SSH em um Sistema Existente}
    
        \begin{enumerate}
            \item Remova o cartão microSD e insira no computador.
            \item Na partição visível chamada \texttt{boot}, crie um arquivo vazio chamado:
            \begin{minted}{bash}
            ssh
            \end{minted}
            (sem extensão).
            \item Opcionalmente, para configurar Wi-Fi, crie também um arquivo \texttt{wpa\_supplicant.conf} com:
            \begin{minted}{bash}
            country=BR
            ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
            update_config=1
        
            network={
                ssid="NOME_DA_SUA_REDE"
                psk="SENHA_DA_SUA_REDE"
            }
            \end{minted}
            \item Recoloque o cartão no Raspberry Pi e ligue o dispositivo.
        \end{enumerate}
    
    \subsection{Verificando o Acesso Bem-Sucedido}
        Ao conectar-se com sucesso, o terminal exibirá:
        \begin{minted}{bash}
        pi@raspberrypi:~ $
        \end{minted}
        A partir desse ponto, o usuário poderá administrar o Raspberry Pi remotamente, instalar pacotes, configurar o VNC e realizar novas etapas do projeto.
        
        \subsection*{Resumo das Situações Comuns}
        \begin{longtable}{|p{5cm}|p{9cm}|}
        \hline
        \textbf{Situação} & \textbf{Ação Recomendada} \\
        \hline
        Sistema liga e lê o SD & Tentar \texttt{ssh pi@raspberrypi.local} (conexão direta) \\
        \hline
        SSH não responde & Criar arquivo \texttt{ssh} na partição \texttt{boot} \\
        \hline
        Rede Wi-Fi disponível & Configurar via arquivo \texttt{wpa\_supplicant.conf} \\
        \hline
        Sistema corrompido ou inacessível & Reinstalar usando o Raspberry Pi Imager \\
        \hline
        \end{longtable}
        
        \subsection*{Próximos Passos}
        Uma vez estabelecido o acesso via SSH, recomenda-se configurar o acesso gráfico via VNC para facilitar o controle remoto do ambiente de desenvolvimento.  
        Isso será abordado no próximo tutorial:  
        \textbf{Tutorial 15 – Acesso Gráfico ao Raspberry Pi via VNC}.

\section{Tutorial 15: Acesso Gráfico ao Raspberry Pi via VNC}
\label{sec: Tutorial 15: Acesso Gráfico ao Raspberry Pi via VNC}

Este tutorial apresenta o procedimento para habilitar o acesso remoto ao ambiente gráfico do Raspberry Pi por meio do \textbf{VNC (Virtual Network Computing)}.  
O VNC permite visualizar e controlar o desktop do Raspberry Pi diretamente do computador, sem necessidade de monitor ou teclado conectados ao dispositivo.

\subsection*{Objetivos}
\begin{itemize}
    \item Ativar o servidor VNC no Raspberry Pi.
    \item Configurar o VNC Viewer no computador para acesso remoto.
    \item Garantir comunicação segura entre o Raspberry Pi e o computador pela rede local.
\end{itemize}

\subsection*{Materiais Necessários}
\begin{itemize}
    \item Raspberry Pi 4 (com sistema operacional funcional e acesso SSH habilitado).
    \item Computador com \textbf{VNC Viewer} instalado (RealVNC ou outro cliente compatível).
    \item Conexão de rede local (via Wi-Fi ou cabo RJ45).
\end{itemize}

\subsection{Ativando o VNC no Raspberry Pi}

\begin{enumerate}
    \item Acesse o Raspberry Pi via SSH:
    \begin{minted}{bash}
    ssh pi@raspberrypi.local
    \end{minted}

    \item Execute o comando de configuração:
    \begin{minted}{bash}
    sudo raspi-config
    \end{minted}

    \item No menu, navegue até:
    \begin{quote}
    \texttt{Interface Options} → \texttt{VNC} → \texttt{Enable}
    \end{quote}

    \item Saia do menu e reinicie o sistema:
    \begin{minted}{bash}
    sudo reboot
    \end{minted}
\end{enumerate}

Após reiniciar, o Raspberry Pi estará executando o serviço VNC.

\subsection{Instalando o VNC Viewer no Computador}

\begin{enumerate}
    \item Acesse o site oficial:  
    \url{https://www.realvnc.com/en/connect/download/viewer/}
    \item Baixe e instale o \textbf{VNC Viewer} compatível com seu sistema operacional.
    \item Após a instalação, abra o programa e, na barra de endereço, digite:
    \begin{quote}
    \texttt{raspberrypi.local}
    \end{quote}
    ou o endereço IP do Raspberry Pi (ex: \texttt{192.168.0.105}).
    \item Quando solicitado, insira as credenciais:
    \begin{quote}
    Usuário: \texttt{pi} \\
    Senha: (a senha configurada no seu sistema)
    \end{quote}
\end{enumerate}

\subsection{Verificando o Acesso ao Desktop}
Se a conexão for bem-sucedida, a tela principal do Raspberry Pi será exibida no computador.  
É possível navegar pelo sistema, abrir aplicativos e até executar o programa do projeto COTB diretamente pelo VNC.

\begin{itemize}
    \item Ícone de rede ou volume podem demorar alguns segundos para carregar — isso é normal.
    \item Caso a tela apareça preta, desconecte e reconecte, pois o VNC às vezes precisa renegociar a resolução.
\end{itemize}

\subsection{Ajustando a Resolução do VNC}
Para evitar distorções na imagem, é recomendável definir uma resolução fixa:

\begin{minted}{bash}
sudo raspi-config
\end{minted}
Em seguida, vá em:
\begin{quote}
\texttt{Display Options → Resolution → 1920x1080}
\end{quote}
Reinicie o Raspberry Pi para aplicar.

\subsection*{Resumo das Situações Comuns}
\begin{longtable}{|p{5cm}|p{9cm}|}
\hline
\textbf{Situação} & \textbf{Ação Recomendada} \\
\hline
Não conecta via VNC & Verificar se o serviço VNC está habilitado e o IP correto está sendo usado \\
\hline
Tela preta após conexão & Desconectar e reconectar o VNC Viewer (ou ajustar a resolução) \\
\hline
Conexão instável & Priorizar o uso do cabo RJ45 em vez de Wi-Fi \\
\hline
\end{longtable}
